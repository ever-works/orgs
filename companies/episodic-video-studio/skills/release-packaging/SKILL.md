# Release Packaging

The package is the episode's first promise to a stranger. Its job is to attract the right viewer honestly — a click that ends in an early exit poisons both the retention signal and the audience's trust.

## Title

- Lead with the episode's genuine dramatic question or stake, sharpened to provoke curiosity — never with a claim the episode does not cash.
- Keep the serialized thread visible: numbering or a consistent naming pattern so returning viewers instantly place the episode in the sequence.
- Draft five, cut to one: the survivor is the title you could defend to someone who just watched the episode.

## Thumbnail concept

- Specify from approved stills: the frame (or framed regeneration request) that captures the episode's key image without spoiling its payoff.
- One focal subject, readable at thumbnail size — composition that works small beats detail that works large.
- Consistent visual identity across the season: a viewer should recognize the show before reading a word.

## Description

- First line does the selling: the premise hook, self-contained, since it is all most surfaces show.
- Then context for the serialized viewer — where this sits in the season, the one-line "previously" — and the standing links between episodes.
- Spoiler rule is absolute: nothing past the setup act is described; the cliffhanger is never referenced, only earned.

## Chapters

- Chapter boundaries follow the actual act structure, named to intrigue rather than summarize — a chapter list is a mini-trailer, and it must not become a spoiler list.

## Publish timing

- Publish per the release calendar: cadence consistency compounds, and an episode's timing should serve the next episode's anticipation.
- Every episode links back to its predecessor and forward (once live) to its successor; the serialized habit is built with plumbing, not hope.

## Honesty audit — run before release lock

- Does the title's promise get kept on screen? Point to the scene.
- Does the thumbnail depict something actually in the episode, in the episode's real tone?
- Does the description spoil nothing past setup?
- Would a viewer who clicked for this package feel the episode delivered it? If any answer wavers, the package fails.

## Output format

One package document per episode: final title, thumbnail spec with source still reference, description text, chapter list with boundaries, publish datetime, and the honesty audit's four answers.
