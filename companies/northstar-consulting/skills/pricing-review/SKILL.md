# Pricing Review

A structured audit that moves from "what should we charge?" to a specific, defensible packaging recommendation. The failure mode it prevents: adjusting price points while the underlying value metric stays misaligned.

## Establish the value metric

1. Identify the unit of consumption that grows as the customer succeeds (seats, records, transactions, workspaces). Write one sentence: "Customers get more value when ___ increases."
2. Test the current pricing against it: does the bill grow with that unit? If not, name the misalignment first — it dominates every other finding.
3. Check the friction direction: does the metric tax adoption (per-seat on a collaboration product) or track it? Note where the metric punishes the behavior the client wants.

## Audit the current packaging

4. Table the current tiers: price, included metric quantity, gating features, and the intended buyer of each tier in one phrase.
5. Find the upgrade walls: for each tier boundary, what actually forces an upgrade? A boundary nothing pushes customers across is decoration.
6. Pull the distribution if available: % of customers per tier, expansion revenue share. A bottom-heavy distribution with no expansion signals a metric or wall problem.

## Compare against the market

7. Build (or reuse the Market Analyst's) competitor table: packaging structure, value metric, public price points, and free-tier posture for 3-5 direct rivals. Date-stamp each row.
8. Position the client: cheaper/premium, same/different metric. A different metric than the whole market needs a stated justification — it is either the strategy or a mistake.

## Model alternatives

9. Design 2-3 alternative structures, each fixing one named problem from steps 2-6. Vary structure (metric, walls, tier count) before varying price points.
10. For each, run the arithmetic: current distribution → plausible migration → revenue delta, with every assumption written and the break-even sensitivity noted.
11. Name the losers per alternative: which segments pay more or lose access, and the grandfathering rule that softens it.

## Quality bar

- The verdict fits in one sentence and names the value metric.
- Every number carries its assumption; every competitor fact carries its date.
- At least one modeled alternative is "change nothing" — the recommendation must beat it explicitly.
- Migration risks and grandfathering are in the body, not a footnote.

## Output format

One-sentence verdict → value-metric analysis → current-packaging audit → competitor table → modeled alternatives with arithmetic → migration risks and rollout note.
