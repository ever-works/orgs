# Plan the MVP

Define the smallest product that lets a real outsider get the core value end to end, and turn it into a build-ready plan.

**Outcome wanted:** a written MVP definition that Nova can fund with confidence and Kai can start building from without a single clarifying question.

**Do:**

- State the one job the MVP does for its first user, in one sentence. Everything in scope must serve that sentence.
- List what is IN (five to eight capabilities, max) and — just as explicitly — what is OUT for v1, with a one-line reason per cut.
- Break the IN list into build tasks, each with a definition of done, ordered so the riskiest assumption is exercised earliest.
- Fold in findings from the "Research the competitor field" task before finalizing: if a competitor gives the core job away free, the plan must say how we differ.
- Name the three assumptions most likely to sink the plan and how the MVP will test each cheaply.

**Definition of done:** the MVP document contains the one-job sentence, the IN/OUT lists with reasons, the ordered build-task breakdown, and the top-three risk assumptions — and Nova has accepted it as the funded scope.
