# Hook Quality Gate

The first seconds of a short vertical video decide whether the rest of it gets watched. This gate is a fixed set of criteria applied to every hook before production. It exists so that "is this opening good enough?" is answered the same way every time, by anyone, and so that failures come back with a named reason instead of a shrug.

## The criteria

A hook passes only if it clears all five:

1. **Specificity.** The opening line could not open any other video. If swapping the topic noun makes it work for a different video, it fails. "This changed everything" fails; a concrete, surprising claim about this topic passes.
2. **Open loop.** The opening creates a question the viewer must stay to resolve. Name the question out loud during review — if you cannot state what the viewer now wants to know, there is no loop.
3. **Honest promise.** The video's body actually resolves the loop. A hook that promises more than the script delivers fails here even if it would retain — burned trust shows up in the channel's numbers within weeks.
4. **Speed.** The loop is open within the first spoken sentence and the first visible frame. Any throat-clearing before the hook — greetings, context, "today we're looking at" — is an automatic fail.
5. **Rotation.** The hook's pattern (its structural shape: bold-claim, question, countdown, contradiction, cold-scene, etc.) differs from the last two published videos and from the adjacent calendar slots. A great hook in a tired shape fails.

## Procedure

1. Label the hook's pattern before judging it. Unlabeled hooks cannot be rotation-checked or analyzed later.
2. Read the opening exactly as it will be heard, at speaking pace, alongside the intended first visual. Judge the combination, not the text alone.
3. Apply the five criteria in order. Stop at the first failure — one named criterion per verdict keeps revisions focused.
4. Record the verdict with the pattern label. Passes carry the label into production and the publish-time metadata ledger; fails go back with the failed criterion and nothing else re-litigated.
5. When retention data shows a passed pattern collapsing at the opening, recalibrate: tighten the criterion that let it through, and write the change down so the gate's history is auditable.

## Quality bar

- Every verdict cites exactly one criterion (the first failure) or states a clean pass.
- No hook enters production ungated, regardless of schedule pressure — a calendar hole is the accepted cost.
- Pattern labels are consistent across the studio; a shared label list is maintained and new patterns are added deliberately, not improvised per review.
