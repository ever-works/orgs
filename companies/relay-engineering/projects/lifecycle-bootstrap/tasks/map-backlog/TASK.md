# Map the backlog into a ranked queue

Turn everything currently claiming to be work — backlog items, open issues, notes,
standing goals on the Work — into the company's first real task queue and cycle plan.

Scope:

- Run the task-discovery playbook over every live source with the CEO: gather raw,
  normalize to one-sentence task statements, deduplicate, and write a definition of
  done for every survivor. Park what cannot be qualified, with reasons.
- Rank the qualified queue on impact, urgency, and effort, and cut it to roughly one
  cycle's worth of committed work.
- Run codebase-recon on the top two candidates so their complexity calls are grounded,
  then produce the first sprint-plan with an explicit committed-versus-stretch line.
- Flag the smallest genuinely useful task in the queue as the candidate for the
  first-ship task.

Deliverable: the ranked queue with definitions of done, the parked list with reasons,
and an accepted first cycle plan. Done when the CEO has accepted the queue, the cycle
plan is published to the delivery team, and the first-ship candidate is agreed.
