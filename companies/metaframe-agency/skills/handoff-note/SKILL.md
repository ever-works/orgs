# Handoff Note

Package work so a cold reader can continue it. The failure mode this prevents: the receiver spending half a day reconstructing what was done, retrying approaches that already failed, or silently redoing finished work.

## Steps

1. Write the note at the moment of handoff, not from memory later — the dead ends and half-decisions evaporate within hours.
2. Structure it in six fixed sections, in this order:
   - **Original ask** — the Task's goal in one sentence, plus any scope changes agreed since, with where each change was agreed.
   - **Done** — what is complete and where it lives (links or paths on the Task/Work). State what "complete" meant: reviewed? drafted? verified how?
   - **Not done** — the remaining work as concrete next actions, ordered, with the very next action first and specific enough to start within five minutes of reading.
   - **Dead ends** — approaches tried and abandoned, each with the reason. This is the highest-value section and the one most often skipped; an unrecorded dead end will be re-dug.
   - **Load-bearing context** — the non-obvious facts the work depends on: constraints discovered, preferences the requester stated verbally, a dependency's quirk, a number that took an hour to find. If losing it would cost more than a minute to rediscover, it belongs here.
   - **Open questions** — decisions the receiver must make or escalate, with your recommendation on each if you have one.
3. Name the receiver and what you expect from them: continue, review, or decide. A note without an addressee is a diary entry.
4. Apply the cold-reader test before posting: could someone with zero conversation history act on this note alone? Every "as discussed" and "the usual place" fails the test — inline the discussion's conclusion and the actual location.
5. Post the note on the originating Task so the chain of custody stays on the record, and notify the receiver.
6. Keep it under a page for a routine handoff; a multi-week effort earns two. Length beyond that means the Done section is narrating process instead of stating results.

## Quality bar

- The next action is startable in five minutes without questions.
- At least one dead end or the explicit line "no approaches were abandoned".
- Zero references that require conversation history to resolve.

## Output format

The six sections in order, addressee and expectation at the top, posted on the originating Task. Bullet points over prose everywhere except load-bearing context, which may need sentences.
